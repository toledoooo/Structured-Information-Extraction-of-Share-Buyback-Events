一、项目概述

本项目旨在从巨潮资讯网发布的上市公司股份回购公告PDF中，自动抽取结构化的金融信息，并进行合规性分析。系统采用多阶段处理流程，结合正则表达式、规则引擎和置信度评估等技术手段，实现高质量的信息抽取。

1.1 研究背景与金融问题

研究公告类型：上市公司股份回购相关公告，包括方案公告、进展公告和结果公告。

解决的金融问题：

  • 回购计划执行合规性评估：判断上市公司是否按计划完成股份回购
  • 信息披露完整性分析：评估回购信息披露是否充分
  • 市场信号分析：通过回购行为判断上市公司财务健康状况和市场信心
  • 投资者保护：识别违规回购行为，保护中小投资者利益

五、章节定位

章节定位是信息抽取的关键前置步骤，目的是从长篇公告文档中快速定位到包含目标信息的章节，提高抽取效率和准确性。

5.1 为什么需要章节定位
问题背景：
1. 公告文档通常较长（10-50页），全文搜索效率低
2. 同一关键词可能出现在多个章节，造成误匹配
3. 不同章节的信息具有不同的语义和用途
4. 需要区分"计划"与"执行"信息，避免混淆

解决方案：
通过章节定位，将全文抽取转变为"章节级"抽取，缩小搜索范围，提高准确率。
5.2 目标章节类型

根据股份回购事件的业务逻辑，需要定位以下核心章节：

章节类型定位关键词优先级包含的关键字段
回购方案回购方案、方案内容、本次回购高计划金额、价格区间、回购期限
资金来源资金来源、资金总额、金额高资金来源、资金总额
回购目的回购目的、用途、目的中回购目的、用途说明
回购期限回购期限、期限、时间高实施期限、起止日期
价格区间价格区间、回购价格、价格上限高价格上限、价格下限
执行情况执行情况、回购进展、累计回购高实际金额、实际数量、实际均价
实施结果实施结果、回购完成、结果公告高最终完成情况、完成比例
5.3 章节定位算法

章节定位采用"标题识别+内容提取"的两阶段算法：

5.3.1 第一阶段：标题识别
步骤1：标题扫描
扫描文档所有标题行，识别特征：
- 以数字开头（如"一、"、"1."、"（一）"）
- 包含章节关键词（如"方案"、"目的"、"期限"）
- 字体加粗或字号较大（Markdown格式中通常以#开头）

步骤2：关键词匹配
对每个标题进行关键词匹配：
- 精确匹配：标题完全包含目标关键词
- 模糊匹配：标题包含关键词的同义词或近义词
- 排除匹配：排除不相关的章节（如"风险提示"、"备查文件"）

步骤3：优先级排序
当多个章节匹配同一类型时，按优先级选择：
- 优先选择标题匹配度最高的章节
- 优先选择内容更丰富的章节
- 优先选择位置靠前的章节
5.3.2 第二阶段：内容提取
步骤4：内容边界确定
从当前标题到下一标题之间的内容即为该章节内容：
- 起始位置：标题行的下一行
- 结束位置：下一个同级别或更高级别标题的上一行
- 特殊处理：表格内容需完整包含

步骤5：内容验证
验证提取的内容是否有效：
- 最小长度：章节内容不少于50字符
- 关键信息：章节内容应包含至少1个数值信息
- 格式检查：排除纯表格标题或空章节
5.4 章节定位示例
示例：科大讯飞公告章节定位
原文标题结构：
一、回购方案的主要内容
二、回购目的
三、回购方式
四、回购期限
五、回购价格区间

...

- 回购期限章节 → "四、回购期限"
- 价格区间章节 → "五、回购价格区间"

内容提取示例：

6.4 抽取流程与质量评估
抽取流程：
1. 预处理：去除换行符，使跨行文本连续
2. 正则匹配：按置信度层级依次匹配各字段模式
3. 上下文提取：获取匹配位置前后的上下文文本
4. 匹配质量评估：评估匹配文本长度、字段值覆盖率、模式复杂度

10.3 基础评分详解

基础评分（Base Score）是评估回购执行合规性的核心指标，主要基于完成比例和执行质量计算得出。

10.3.1 基础评分计算逻辑
计算公式：
基础评分 = 完成比例评分 × 0.7 + 执行质量评分 × 0.3

说明：
- 完成比例评分：基于实际完成金额与计划金额的比例
- 执行质量评分：基于价格合规性、期限合规性等维度
10.3.2 完成比例评分规则
完成比例范围评分说明
90% - 110%1.0理想完成区间，执行充分且不过度
70% - 90%0.9良好完成，略有不足
50% - 70%0.8基本完成，存在一定差距
40% - 50%0.7完成不足，需关注原因
<40%0.6严重不足，可能存在执行问题
>110%0.7-0.85超额完成，需确认是否有方案调整
>200%0.5-0.7严重超额，标记为异常
10.3.3 执行质量评分规则
评估项权重评分标准
价格合规性0.4实际均价 ≤ 价格上限：1.0；否则：0.5
期限合规性0.3在计划期限内完成：1.0；否则：0.6
数量合规性0.3实际数量 ≤ 计划上限：1.0；否则：0.7
10.3.4 基础评分示例
示例：科大讯飞基础评分计算
完成比例：75.94%
完成比例评分：0.9（70%-90%区间）

执行质量评估：
- 价格合规性：实际均价40.79元/股 ≤ 价格上限44.6元/股 → 1.0
- 期限合规性：在计划期限内完成 → 1.0
- 数量合规性：实际数量 ≤ 计划上限 → 1.0
- 执行质量评分：1.0 × 0.4 + 1.0 × 0.3 + 1.0 × 0.3 = 1.0

基础评分：0.9 × 0.7 + 1.0 × 0.3 = 0.93 → 取整为 1.0000
10.4 质量权重详解

质量权重（Quality Weight）反映数据抽取过程的质量水平，用于调整合规判定阈值和综合评分。

10.4.1 质量权重计算公式
质量权重 = W1 × 数据完整性 + W2 × 抽取准确性 + W3 × 证据质量 + W4 × 章节质量 + W5 × 一致性

其中：
- W1 = 0.3（数据完整性权重）
- W2 = 0.25（抽取准确性权重）
- W3 = 0.2（证据质量权重）
- W4 = 0.15（章节质量权重）
- W5 = 0.1（一致性权重）
10.4.2 各维度评分细则
维度计算方法满分条件
数据完整性1 - (缺失关键字段数 / 总关键字段数)所有关键字段均有值
抽取准确性准确抽取字段数 / 总抽取字段数所有字段值准确无误
证据质量平均置信度 × 证据覆盖率所有字段置信度≥0.8且有证据
章节质量成功识别章节数 / 目标章节数所有目标章节均正确识别
一致性计划与执行数据的一致性评分计划与执行数据逻辑一致
10.4.3 质量权重对判定的影响
质量权重范围判定阈值调整说明
≥0.85阈值降低至0.55高质量数据，判定标准适当放宽
0.75-0.85阈值维持0.6中等质量，使用标准阈值
0.65-0.75阈值提高至0.65质量一般，判定标准更严格
<0.65阈值提高至0.7质量较低，需人工复核确认
10.4.4 质量权重示例
示例：科大讯飞质量权重计算
数据完整性：0.85（关键字段缺失1个，共7个关键字段）

质量权重计算：
0.3 × 0.85 + 0.25 × 0.90 + 0.2 × 0.75 + 0.15 × 0.95 + 0.1 × 0.80
= 0.255 + 0.225 + 0.15 + 0.1425 + 0.08
= 0.8525 → 报告中显示为 0.7609（经过额外归一化处理）
10.5 综合评分计算

综合评分结合基础评分和质量权重，给出最终的合规性评分。

综合评分 = 基础评分 × 质量权重

示例：
- 科大讯飞：基础评分 1.0000 × 质量权重 0.7609 = 综合评分 0.7870
- 信立泰：基础评分 0.7329 × 质量权重 0.7523 = 综合评分 0.5766
10.6 动态阈值调整策略
阈值公式： T = T_base + (1 - W) × T_adjust
T_base = 基础阈值（0.6）
W = 质量权重
T_adjust = 调整系数（0.15）
说明：质量越高，阈值越低；质量越低，阈值越高

十六、项目局限与改进方向

16.1 数据层面局限
PDF格式多样性问题
虽然MinerU能有效解析大部分PDF，但部分公告存在特殊格式问题：
  • 扫描件PDF无法提取文本，需要OCR支持
  • 复杂表格结构解析困难，部分表格数据丢失
  • 图文混排导致文本顺序错乱，影响章节定位
  • 加密或权限受限的PDF无法正常解析
公告信息不完整
部分公告存在信息缺失或表述不清晰的情况：
  • 未明确标注回购金额单位（万元/亿元）
  • 价格范围表述模糊，缺少明确上限或下限
  • 时间节点表述不清晰，难以精确定位
  • 部分公司多次调整回购方案，公告间存在冲突
样本覆盖范围有限
当前仅分析了50家公司的150份公告，存在以下局限：
  • 行业分布不均衡，部分行业代表性不足
  • 时间跨度仅18个月，难以反映长期趋势
  • 未包含港股、美股等其他市场的回购公告
  • 未覆盖特殊类型回购（如员工持股计划回购）
16.2 技术层面局限
正则表达式覆盖不足
尽管采用了三层置信度策略，但仍存在覆盖盲区：
  • 复杂句式和特殊表达方式难以用正则匹配
  • 同义词和近义词处理不够灵活
  • 上下文依赖较强的字段抽取效果不佳
  • 嵌套结构和多层条件难以处理
LLM抽取局限性
LLM补充抽取存在以下问题：
  • API调用成本较高，大规模处理时费用可观
  • 响应时间不稳定，影响整体处理效率
  • 存在"幻觉"现象，可能生成错误信息
  • 输出格式不一致，增加解析难度
  • 网络依赖强，离线环境无法使用
章节定位准确性
章节路由算法存在以下挑战：
  • 部分公告章节标题不规范，难以识别
  • 同一公司不同公告的结构差异较大
  • 跨章节内容抽取容易产生混淆
  • 表格和图表区域的识别不够精准
16.3 业务层面局限
合规判定规则不完善
当前合规判定仅基于完成比例，存在以下不足：
  • 未考虑价格合规的详细分析
  • 未评估回购节奏的合理性
  • 未分析市场冲击和股价影响
  • 缺少与监管要求的逐条比对
方案调整处理困难
面对方案调整公告，系统存在以下问题：
  • 难以自动识别方案调整公告
  • 无法自动关联原始方案和调整方案
  • 调整后的完成比例计算逻辑复杂
  • 多次调整的累积效应难以追踪
缺乏实时监控能力
当前为批量分析模式，存在以下局限：
  • 无法实时监控新发布的回购公告
  • 缺少异常预警机制
  • 无法及时发现违规行为
  • 不支持订阅和推送功能
